Header

The header of the request contains parameters to authenticate and authorize the request. The request is authenticated only if the API authentication is enabled in the tenant configuration. See Authentication Services, for more information. To authenticate the request, you must:
  • Use https and append the request with the current time-stamp in ISO_8601 format (&timeStamp="current timestamp in ISO_8601 format")
  • Generate authToken by hashing the request using the clientAuthKey with HmacSHA256 algorithm.
  • Pass the clientId, userId, and authToken in the header.

You can also pass additional parameter such as ownershipData and userRoles in the request header to authorize the request. See Authorization Model and Understanding the Request Headers, for more information. {% endif %} {% if site.build != "internal" %}
