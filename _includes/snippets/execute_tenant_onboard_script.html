1. Login to one of the manager node.

    2. Result: "home" directory of manager node is displayed. You can see the "tenant_onboarding.sh" file present.

    {% picture tenant-onboard_n.png alt="tenant_onboard.sh File" %}
  2. The "tenant_onboarding.sh" does the following activities:
    • Creates elastic search indices with custom mapping.
    • Applies templates and executes the scripts to configure Elasticsearch.
    • Creates aliases for indices.
    • Creates HDFS folders.
    • Creates HBase tables.
    • Starts storm topologies.
    • Submits topologies based on the value provided for SUBMIT_TOPOLOGIES in the tenant_onboarding.sh script.
  3. Depending on your requirement, update the "tenant_onboarding.sh" script by providing the values for following keys:
    • RDP_INSTANCE_ID: Provide the instance name in the "rdp_'ID'" format.
    • DEPLOY_VERSION: Provide "rdp_deploy" version.
    • STACK_TYPE: Provide the the details for which operation this deployment is intended. The suggested values are dev, prod, and qa.
    • TENANT_ID: Provide the required tenant name.
    • SUBMIT_TOPOLOGIES: Provide the value as "1" for the fresh deployment. If you are creating new tenant on existing instance, provide the value as "0".
    4. {% picture tenant-onboard-sh_n.png alt="Parameters for tenant-onboard.sh File" %}
  4. Execute the script tenant_onboarding.sh by issuing ./tenant_onboarding.sh command.
